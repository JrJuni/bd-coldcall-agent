필수 규칙:
- 제품과 명확히 핏이 맞는 산업을 정확히 {n_industries} 개, 핏 강도 순으로 선정 (가장 좋은 것부터).
- 각 산업마다 실제로 존재하는 회사를 정확히 {n_per_industry} 개 나열. 널리 알려진 이름 (예: NVIDIA, Snowflake) 만 사용하고, 회사명을 절대 창작하지 마세요. 존재 확신이 없으면 다른 회사로 교체.
- 각 회사에 티어 부여:
    - S = direct trigger (즉시, 고예산 핏; 대표 로고)
    - A = strong fit (명확한 use-case 매칭; 중대형 엔터프라이즈)
    - B = adjacent signal (괜찮은 핏; 추가 디스커버리 필요)
    - C = long-shot (투기적; 우선순위 낮은 아웃리치)
- 티어 분포 — 모든 회사를 S 로 또는 C 로 도배 금지. 근거가 있는 만큼 섞으세요.
- 각 회사의 `rationale` 은 1~2문장, 이 제품이 이 회사에 구체적으로 왜 핏인지 설명 (지식 베이스에서 끌어와야 하며, 일반론 금지).
- `industry_meta` rationale 은 산업 전체에 왜 이 제품이 핏인지 1문장.
- 모든 출력 텍스트 (산업명, 회사명, rationale) 는 지식 베이스 언어와 무관하게 한국어로 작성.

출력 형식:
- JSON 만. 서문·마크다운 헤더·코드 펜스 금지.
- 정확히 두 개의 top-level 키를 가진 단일 JSON 객체:
    - `industry_meta`: 산업명 → 1문장 rationale 매핑 객체.
    - `candidates`: `name`, `industry`, `tier`, `rationale` 키를 가진 객체의 평면 배열.
- 모든 `candidate.industry` 값은 `industry_meta` 키 중 하나와 정확히 일치해야 함.
- `candidates` 배열은 정확히 {n_industries} × {n_per_industry} = {expected_total} 개 항목, 산업당 정확히 {n_per_industry} 개.
- `tier` 는 "S", "A", "B", "C" 중 정확히 하나.
